# Jamming Detector Handheld - AI Agent Rules & Project Boundaries

ไฟล์นี้เป็นข้อกำหนด กฎเกณฑ์ และขอบเขตทางวิศวกรรมสำหรับ **AI Coding Agent** และนักพัฒนาทุกคนในการทำงานกับโปรเจกต์เครื่องตรวจจับสัญญาณรบกวนจีพีเอสแบบพกพานี้ เพื่อความเสถียรของระบบ ป้องกันความเสียหายทางฮาร์ดแวร์ และรักษาประสิทธิภาพของบอร์ดจำกัดทรัพยากร (Raspberry Pi Zero 2 W)

---

## 1. ปรัชญาหลักในการทำงาน (Core Philosophy)
* **อย่าเปลี่ยนอะไรที่ไม่เกี่ยวข้องกับงาน:** ยึดมั่นกฎ "Minimal Invasive" แก้ไขเฉพาะโค้ดจุดที่ได้รับคำสั่งโดยตรง ห้ามลบ/ปรับเปลี่ยนฟังก์ชันอื่นๆ หรือเปลี่ยนการทำงานหลักของระบบหากผู้ใช้ไม่ได้สั่ง
* **ห้ามใช้ Placeholder:** โค้ดและข้อมูลทดสอบจะต้องทำงานได้จริง ห้ามเขียนคอมเมนต์ละเว้น เช่น `// TODO: implement later`
* **เคารพขีดจำกัดฮาร์ดแวร์:** ตระหนักเสมอว่าโปรเจกต์นี้ทำงานบน Raspberry Pi Zero ซึ่งประมวลผลสัญญาณวิทยุ (DSP) ที่หนักหน่วงอยู่แล้ว ทุกการประมวลผลและการจัดเก็บข้อมูลต้องมีประสิทธิภาพสูงสุด

## 2. ขอบเขตและกฎความปลอดภัยทางฮาร์ดแวร์ (Hardware & Safety Boundaries)

### ⚠️ กฎเหล็กเรื่องแรงดันไฟฟ้าและพอร์ต I2C
* **I2C Address & AD0 Pin:** 
  * บอร์ด GY-9250 และ RTC (DS3231) แชร์บัส **I2C1 (SDA: Pin 3, SCL: Pin 5)** ร่วมกัน
  * **ที่อยู่เริ่มต้น (Default Address):** RTC อยู่ที่ `0x68`; GY-9250 ต้องตั้งให้เป็น `0x69` โดยผูกขา AD0/ADO เข้ากับ **3.3V**
  * **ข้อห้ามสำคัญที่สุด:** **ห้ามป้อนไฟ 5V เข้าขา AD0/ADO ของ GY-9250 โดยเด็ดขาด!** ให้ใช้ 3.3V เท่านั้น เพื่อเลี่ยง I2C address collision กับ DS3231 และป้องกันความเสียหายกับขา I/O ของ IMU
* **การจ่ายไฟและหลีกเลี่ยง Under-voltage:**
  * อุปกรณ์ RTL-SDR Dongle กินกระแสไฟสูงมาก (300-500mA) รวมกับจอ TFT LCD และชิป Wi-Fi 
  * ห้ามเขียนโปรแกรมที่ดึงโหลด CPU ขึ้น 100% ต่อเนื่องอย่างไร้จุดหมาย และห้ามตั้งเวลาเขียนข้อมูลลง SD Card บ่อยเกินไป เพราะความต้านทานแรงดันจะตกลงและทำให้ Wi-Fi หลุด

### 🔌 กฎการตั้งค่า SPI (จอภาพและทัชสกรีน)
* จอภาพ ILI9488 และระบบทัชสกรีน XPT2046 แชร์พอร์ต **SPI0** ร่วมกัน โดยแยกขาเลือกใช้งานผ่าน CS:
  * **LCD_CS:** GPIO 8 (Pin 24)
  * **T_CS:** GPIO 22 (Pin 15)
  * ห้ามแก้ไขค่าพิน SPI หรือวิธีส่งข้อมูลภาพในโค้ดหากไม่จำเป็น เนื่องจากจะส่งผลให้จอแสดงผลเป็นสีขาวหรือเกิดการแตะรัวที่ตำแหน่งทัชสกรีน

## 3. กฎการจัดการฐานข้อมูล (Database & I/O Guidelines)
เพื่อยืดอายุการใช้งาน MicroSD Card และป้องกันการสะดุดดีเลย์ของหน้าจอประมวลผลหลัก (GUI):
* **ห้ามบันทึกข้อมูลถี่กว่าความจำเป็น (Adaptive Logging):**
  * ห้ามสั่งเขียนฐานข้อมูล SQLite `log_event` ทุกๆ 1 วินาที
  * **SCANNING State:** บันทึกข้อมูลประวัติ (Heartbeat) ห่างๆ ทุกๆ **30 วินาที**
  * **WATCH / JAMMING State:** บันทึกข้อมูลถี่ขึ้นเป็นทุกๆ **3 วินาที**
  * **State Changed (การเปลี่ยนสถานะ):** ให้ทำการบันทึกข้อมูลลงฐานข้อมูล **ทันที** เพื่อไม่ให้พลาดจุดเริ่มต้นและจุดสิ้นสุดของเหตุการณ์กวนสัญญาณ
* **จำกัดขนาดฐานข้อมูล (Pruning):**
  * รักษาฟังก์ชันลบแถวประวัติที่เกิน 1,000 แถวแรกไว้เสมอ เพื่อให้ฐานข้อมูลมีขนาดเล็กและเปิดใช้งานได้เร็วบนแรมของ Pi

## 4. กฎการพัฒนาเว็บอินเทอร์เฟซ (Web Dashboard & API)
* **การลดภาระคอขวดบนบัสสื่อสาร:**
  * หน้าจอเว็บและ API ของ Flask จะต้องทำงานแยกเธรดจากลูปการประมวลผลสัญญาณคลื่นความถี่หลักเสมอ
  * หน้าเว็บบ้านดึงข้อมูลจากแรมโดยตรงผ่าน `/api/status` โดยห้ามตั้งค่าการดึงบ่อยกว่า **500ms (POLL_MS = 500)** เพื่อไม่ให้ CPU โดนขัดจังหวะการทำงานมากเกินไป
  * ข้อมูลอาเรย์คลื่นความถี่ (Spectrum Array) ที่ส่งออกไปหาเว็บ จะต้องลดขนาดลงเหลือน้อยกว่า 240 จุดเสมอ (Downsampling) เพื่อให้ Payload ทางเครือข่ายมีขนาดเล็กและประมวลผลฝั่งไคลเอนต์ลื่นไหล
* **งานดีไซน์หรูหราพรีเมียม (Rich Aesthetics):**
  * รักษาโทนสีที่พรีเมียม สไตล์เรียบหรูโทนสีเข้ม (Dark Mode / Glassmorphism) และใช้ตัวแปร CSS ร่วมกันเพื่อปรับธีมตามสถานะแจ้งเตือน (เขียว-Scan, เหลือง-Watch, แดง-Jamming)

## GY-9250 / MPU9250 hardware note

- This project uses GY-9250 as the permanent IMU module.
- Keep `IMU_MODEL = 'GY-9250'` and `IMU_ADDRESS = 0x69` when DS3231 RTC remains on the same I2C1 bus at `0x68`.
- Tie the GY-9250 AD0/ADO pin to **3.3V only** before booting. Do not connect AD0/ADO to 5V.
- The app uses the GY-9250 gyro for the current polar radar bearing. The AK8963 magnetometer is initialized for diagnostics and future true-heading use, but the main field UI still uses the gyro-relative bearing path.
